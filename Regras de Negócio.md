| 8                      | Consulta de usuário autenticado retorna id e username se encontrado.                                     | token JWT                         | 200 - { id, username }                                                 |

| 15                     | Só é possível atualizar status se for um valor válido.                                                   | status                            | 400 - "Status inválido"                                                |
| 16                     | Remoção de destino só é permitida se o destino existir e pertencer ao usuário.                           | userId, id                        | 200 - "Destino removido com sucesso" ou 404 - "Destino não encontrado" |
| 17                     | Todos os campos opcionais de destino (details, date) são preenchidos com valor padrão se não informados. | details, date                     | details: "", date: null (default)                                      |
| 18                     | Autenticação JWT é obrigatória para todas operações de destino.                                          | token JWT                         | 401 - "Token não fornecido" ou 403 - "Token inválido"                  |
